MineSight: Vision-Augmented Minecraft
System
Updated Technical Specification — June 2026
Overview
MineSight is a real-time AI-powered perception system for Minecraft. It separates responsibilities between a
lightweight client-side mod and an external Python-based machine learning engine. The system detects ores
and other objects using computer vision and overlays that information back into the game, with support for
tracking, memory, and optional 3D world mapping.
Core Concept
(cid:127) The Minecraft mod does not perform detection
(cid:127) The Python backend performs all ML inference
(cid:127) Communication is handled via WebSockets
(cid:127) The system operates in real-time

Why YOLO26s for This Project
(cid:127) YOLO26s has enhanced small-object detection via ProgLoss and STAL loss functions — critical for
small/occluded ores
(cid:127) NMS-free end-to-end architecture reduces post-processing latency
(cid:127) RTX 4060 Ti runs YOLO26s at 100+ FPS at 640 resolution — well within the 30 FPS target
(cid:127) Higher accuracy than nano without meaningful performance cost on target hardware
(cid:127) Pretrained weights available: yolo26s.pt from Ultralytics assets

Data Collection Strategy
(cid:127) Primary source: Roboflow Universe (existing Minecraft ore datasets)
(cid:127) Supplement with manual screenshots at varied brightness/gamma (cave lighting variance)
(cid:127) Intentionally oversample rare ores (diamond, emerald) — rarity in-game causes class imbalance
(cid:127) Include partially occluded ores — fully exposed examples alone will cause real-world failures
(cid:127) Capture at multiple FOV values (matches FOV field sent in player state messages)
Roboflow Augmentations (apply before export)
(cid:127) Brightness/exposure variation — critical for cave vs. surface lighting
(cid:127) Horizontal flip — free data, ores are symmetrical
(cid:127) Random crop and zoom — simulates varied viewing distances
(cid:127) Mosaic augmentation — also supported natively by YOLO26 during training

Screen to World Mapping (Advanced)

Goal: Convert 2D detections into 3D world coordinates
Inputs:
(cid:127) Player position
(cid:127) Camera rotation (yaw, pitch)
(cid:127) Field of view (FOV)
(cid:127) Screen resolution
Method:
1. Normalize screen coordinates
2. Convert to camera space ray
3. Apply rotation transforms
4. Raycast into world
Output: Approximate 3D position of detected object
This is the most complex component. Depth estimation from 2D bounding box centers introduces real-world error. Flag
this in implementation and iterate.
